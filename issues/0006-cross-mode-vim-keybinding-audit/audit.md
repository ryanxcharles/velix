# Cross-mode Vim Keybinding Audit

This audit records bindings by mode. A key that works in normal mode is not
considered working in select mode unless select-mode behavior has separate
evidence.

Status meanings:

- `Works`: verified against the current Vim profile without code changes in this
  experiment.
- `Fixed`: this experiment changed Velix and added verification.
- `Different by design`: verified, but intentionally follows Helix/Velix
  semantics.
- `Unsupported for now`: no compatible Velix behavior exists in this slice.

## Follow-up Audit Areas

Experiment 1 intentionally fixes a first select-mode batch while creating the
mode-specific audit structure. Follow-up experiments must continue through the
remaining common Vim bindings by mode, including broader visual/select editing,
operator-pending grammar, registers, marks, command-line behavior, search prompt
behavior, paste/register metadata, and insert-mode control keys.
