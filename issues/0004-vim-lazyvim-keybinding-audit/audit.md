# Vim and LazyVim Keybinding Audit

This table is the first Issue 4 audit slice. It covers normal-mode bindings that
are either basic Vim muscle memory or common LazyVim workflow aliases already
targeted by the Velix Vim profile.

Status meanings:

- `Works`: verified against the current Vim profile without code changes in this
  experiment.
- `Fixed`: this experiment changed Velix and added verification.
- `Different by design`: verified, but intentionally keeps Helix/Velix
  semantics.
- `Unsupported for now`: no compatible Velix behavior exists in this slice.
